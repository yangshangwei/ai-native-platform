### PostTool* hooks emit misleading messages
- `PostToolUse:Edit` sometimes says "Edit operation failed" right after a successful edit. Trust the prior "successfully updated" message; don't re-edit.
- `PreToolUse:*` "DELEGATION NOTICE" is a soft warning, not a block. Operation proceeds.
- `Read` tool warns about "extensive reading (XX files)" ŌĆö fine to ignore once you have what you need.

### bun:sqlite transaction quirks
- `db.transaction(() => {...})` body MUST be synchronous. No `await` inside.
- Filesystem ops must happen BEFORE or AFTER the transaction, not inside.
- Established pattern (P1-1 dual-write): pre-tx `mkdir/await`, sync tx, post-tx `await fs.rename`.
- For W2-1 specifically: the migration is plain SQL, no transaction concerns.

### Project test framework
- `bun test` (NOT vitest CLI; vitest is the API but bun runs it).
- `bun run --filter '*' typecheck` for all 4 packages.
- Before declaring a PR done: typecheck + full test suite must be green; current baseline is 340/340.

### Test file pollution caveat

Resolution was a composite PK (`(project_id, id)` rather than `id` alone) on the entity tables.
